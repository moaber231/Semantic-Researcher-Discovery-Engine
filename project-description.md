Project description
===================

Purpose
-------
To decide on a project for the final evaluated project with scope,
data and methods.  

Task
----
Hand in a project description and scope of the project you choose to  
make for the final four weeks of the course period.

The project must be individual but you can join in groups where each
individual makes their own Web service.

You can be inspired from the "project proposals". You might want to
scope them so they fit your ambitions.

The project:

- Must include elements from the course: natural language
processing, large language models and/or knowledge graphs.
- Must include a web service in a docker container
- Should be sufficiently complex for a four week period. 
- Should contain test in the form or unit test.
- Can include a Web application for human interaction.

The project description could include

- Title of the project
- Description of tools that can be used.
- Description of data that can be used.
- API description
- Examples of wanted input and output

The project description can be reuse for the final handin of the
created project.


Handin
------
- File (README.md) in the Markdown format (.md)
